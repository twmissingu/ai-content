# 内容类型体系全景

> 供 AI Agent 快速参考的完整内容分类、结构、风格指南。
> 人设规范：[persona_bible.md](../config/prompts/persona_bible.md)
> 提示词模板目录：`config/prompts/`
> 风格配置：`config/writing_styles.json`

---

## 一、类型判定决策树

```
内容涉及几个产品/事件？
├── 1 个
│   ├── 有跑分/价格/对比？ ────────────→ 工具评测类
│   ├── 有操作步骤可跟练？ ────────────→ 教程类
│   ├── 有个人独立改进？ ──────────────→ 分享类
│   ├── 有框架/方法论/等级？ ──────────→ 经验总结类
│   ├── 是概念的历史/发展/意义？ ──────→ 技术科普类
│   └── 是对现象的独立观点/思辨？ ─────→ 观点评论类
├── 3-8 个同类事物 ────────────────────→ 盘点合集类
└── 10+ 个产品/事件（发布会/大会） ───→ 新闻类
```

## 三、每种类型的标题公式

### 1. 分享类

| 公式 | 示例 |
|------|------|
| 我发现了一个{形容词}的{东西} | 我发现了一个记住任何概念的方法 |
| 这个{N}K 星标的开源神器 | 这个 51K 星标的开源神器，太好用了 |
| 试了{X}之后，我觉得{观点} | 试了 Cursor 之后，我觉得 VS Code 该退役了 |
| {时间}，{产品}终于{动作}了 | 等了两年，Claude 终于能联网了 |

### 2. 技术科普

| 公式 | 示例 |
|------|------|
| 一文带你看懂{概念}到底是个啥 | 一文带你看懂 MCP 到底是个啥 |
| {产品}终于支持{功能}了，但这背后更有意思 | ChatGPT 终于支持记忆了，但这背后更有意思 |
| 你每天都在用的{技术}，可能不知道它叫什么 | 你每天都在用的 Markdown，可能不知道它叫这个名字 |
| 一个{人物}的{故事}，改变了整个{领域} | 一个 17 岁少年的代码，改变了整个互联网 |

### 3. 教程类

| 公式 | 示例 |
|------|------|
| {产品}全面教程：从安装到{高阶用法}，一篇搞定 | Claude Code 全面教程：从安装到多 Agent 协作，一篇搞定 |
| 手把手教你用{产品}，{做出什么} | 手把手教你用 Codex，10 分钟做出一个网页 |
| {产品}最强教程：{竞品}用户转过来必看 | Codex 最强教程：Claude Code 用户转过来必看 |

### 4. 新闻类

| 公式 | 示例 |
|------|------|
| {公司}刚刚发布了{N}个新东西，我帮你整理完了 | Google 刚刚发布了 20 多个新东西，我帮你整理完了 |
| {大会}：一句话总结，{判断} | Google I/O：一句话总结，Agent 才是主角 |
| 可能是最全的{大会}整理：{亮点} | 可能是最全的 WWDC 整理：AI 全面入侵 |

### 5. 经验总结

| 公式 | 示例 |
|------|------|
| 观察了{时间}，我把{领域}分成了{N}个等级 | 观察了两年，我把 AI 用户分成了 5 个等级 |
| 做了{时间}，我沉淀下来的{N}条经验 | 做了三年内容，这 3 条经验最值钱 |
| 关于{领域}，我想通了一件事 | 关于做内容，我想明白了一件事 |
| {领域}最重要的不是{大众认知}，而是{洞察} | 做内容最重要的不是写作，而是找角度 |
| 如何在{时代}，找回你被{动词}的{能力} | 如何在 AI 时代，找回你被削弱的思考力 |

### 6. 工具评测

| 公式 | 示例 |
|------|------|
| 实测{产品}，{一句话定性} | 实测 DeepSeek V4，国产模型终于能打了 |
| {产品}vs{产品}，{结论} | DeepSeek V4 vs MiMo，谁更值得接 Claude Code？ |
| {场景}最佳选择，我选{产品} | Claude Code 接国产模型，我现在最推荐这个 |
| {产品}{意外特点}，没想到 | Google 和 OpenAI 居然合作了，没想到 |

### 7. 观点评论（独占公式加粗）

| 公式 | 示例 |
|------|------|
| **因为{事件}，整个{领域}都变成了{比喻}** | **因为 GPT-image-2，整个互联网都变成了黑暗森林** |
| {领域}正在{趋势}，但大多数人还没意识到 | AI 编程正在吞噬一切，但大多数人还没意识到 |
| 关于{事件}，我想说点真话 | 关于 AI 替代人类，我想说点真话 |

### 8. 盘点合集（独占公式加粗）

| 公式 | 示例 |
|------|------|
| **分享{N}个我觉得{场景}必装的{东西}** | **分享 6 个我觉得用 Claude Code 必装的 Skills** |
| {N}个让我{感受}的{东西} | 5 个让我效率翻倍的 AI 工具 |

## 四、每种类型的核心结构

### 1. 分享类

**微信版：**
```
个人故事 → 问题/痛点 → 发现 → 原版介绍 → 独立思考改进 → 演示效果 → 升华
```

**小红书版：**
```
钩子 → 是什么 → 怎么用 → 我的改进 → 试了效果 → 感悟 → 互动
```

**独占要求：** 必须有"我的改进/优化"层，不能纯搬运。

---

### 2. 技术科普

**微信版：**
```
产品钩子 → 概念解释 → 起源故事 → 发展历程 → 核心论点 → 思辨 → 升华
```

**小红书版：**
```
冷知识钩子 → 是什么 → 谁发明的 → 为什么现在火了 → 一个好玩的事 → 一句话总结 → 互动
```

**独占要求：** 必须有时间线叙事 + "为什么现在火了"的时代关联。

---

### 3. 教程类

**微信版：**
```
引言 → 安装 → 界面 → 设置 → 插件/扩展 → 实战入门 → 实战进阶 → 收尾
```

**小红书版：**
```
钩子 → 是什么 → 怎么装 → 界面 → 必改设置 → 亮点功能 → 跟我做一个 → 互动
```

**独占要求：** 必须有两轮实战（入门 + 进阶），不能只讲安装。

---

### 4. 新闻类

**微信版：**
```
开场定性 → 分类板块（模型/产品/Agent/电商/其他）→ 每个板块含评价 → 收尾升华
```

**小红书版：**
```
钩子 → 3-5 个最有看点的要点（每个 2-3 行）→ 互动
```

**独占要求：** 必须分类板块 + 评价分级，不能流水账。小红书版只选最有看点的 3-5 个，不全量覆盖。

---

### 5. 经验总结

**微信版：**
```
个人故事 → 底层认知 → 方法论框架（2-4步）→ 逐步展开 → 底线原则 → 数据指标 → 收尾
```

**小红书版：**
```
核心洞察 → 3-5 条经验（每条 2-3 行）→ 底线 → 互动
```

**独占要求：** 必须有底层认知 + 方法论框架，不能只列经验。

---

### 6. 工具评测

**微信版：**
```
时效钩子 → 跑分数据 → 真实体验 → 价格分析 → 适用人群 → 行业展望
```

**小红书版：**
```
钩子 → 跑分 → 价格 → 真实体验 → 适用人群 → 行业展望 → 互动
```

**独占要求：** 必须有跑分/数据 + 价格 + "谁该用谁不该用"。

---

### 7. 观点评论

**微信版：**
```
反差钩子 → 观察与现象 → 核心论点（公理/框架推导）→ 跨领域类比 → 升华
```

**小红书版：**
```
反差钩子 → 核心论点 → 3-5 个论据 → 升华 → 互动
```

**独占要求：** 必须有公理/框架推导 + 跨领域类比 + 升华到哲学/社会/人性层面。

---

### 8. 盘点合集

**微信版：**
```
痛点铺垫 → 筛选标准 → 逐个推荐（每个 300-500 字，含体验+优缺点）→ 使用建议
```

**小红书版：**
```
钩子 → N 个推荐（每个 emoji + 名称 + 一句话推荐理由）→ 互动
```

**独占要求：** 必须有筛选标准 + 推荐排序，不能无序罗列。

## 五、微信 vs 小红书核心差异

| 维度 | 微信公众号 | 小红书 |
|------|-----------|--------|
| 阅读场景 | 深度阅读，10-15 分钟 | 碎片浏览，30 秒-2 分钟 |
| 内容策略 | 全面覆盖，层层递进 | 挑最值得说的 3-5 个点 |
| 结构 | 长叙事 + 方法论 + 案例 | 条目式，每条 2-3 行 |
| 语言 | 可以有长句、复杂逻辑 | 每句 ≤25 字，口语化 |
| 互动 | 文末引导 | 每条要点独立可截图传播 |
| 配图 | 0-10 张（文字为主） | 3-6 张（封面图决定 80% 点击率） |
| 结尾 | 统一功能性结尾（点赞+在看+转发） | 互动问题 + 收藏引导 |

## 六、统一人设规范（摘要）

完整定义见 [persona_bible.md](../config/prompts/persona_bible.md)。

**身份：** 九卿，AI 工具重度用户和观察者。不是技术布道者，不是培训讲师。

**语气：**
- 第一人称"我"叙事
- 口语化，像跟朋友聊天
- 有观点，敢下判断（"坦率的讲"、"我觉得"）
- 有真实吐槽（不全夸）
- 有数据锚点（具体数字）
- 金句密度：每 300-500 字至少一句可截图传播的话

**禁止用词：** 赋能、生态、闭环、据悉、据了解、让我们一起来看看、今天给大家分享一下

**功能性结尾（微信）：**
```
---
以上，觉得不错，随手**点赞 + 在看 + 转发**三连吧。
{类型相关的互动问题}
```

**功能性结尾（小红书）：**
```
💬 {互动问题}
📌 收藏这篇，{什么时候翻出来看}
```

## 七、XHS 模板通用规范

所有小红书模板共享以下约束：

| 规范 | 要求 |
|------|------|
| 单句长度 | ≤25 字 |
| Emoji 密度 | 连续 3 行必须有 Emoji 或加粗 |
| 封面图 | 1080×1440px（3:4 竖版） |
| 配图数量 | 3-6 张 |
| 结尾 | 互动问题 + 收藏引导 |
| 功能性 Emoji | 每个要点 1 个，用于视觉分隔 |
| 语气 | 口语化、感叹、短句 |

**禁止使用：**
- "让我们来看看"、"今天给大家分享"
- 超过 30 字的单句
- 连续 3 行没有 Emoji 或加粗
- 学术化定义
- 纯参数堆砌（必须有结论/评价）

## 八、高互动文章的共同特征

基于 13 篇真实文章数据验证：

| 特征 | 说明 | 互动效果 |
|------|------|---------|
| 反差开头 | 第一句话制造认知冲击 | 阅读完成率 ↑ |
| 跨领域类比 | 用游戏/影视/生活降低理解门槛 | 评论区主动引用 |
| 框架/分级 | "我把 X 分成了 N 个等级" | 驱动转发（自我定位） |
| 金句密度 | 每 300-500 字一句可截图传播的话 | 截图分享 ↑ |
| 数据锚点 | 具体数字（价格、跑分、百分比） | 可信度 ↑ |
| 真实吐槽 | 不全夸，敢说缺点 | 信任感 ↑ |
