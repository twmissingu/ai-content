# 技术科普 — 微信公众号版

> 适配"九卿"人设，发布平台：微信公众号
> 继承排版规范：[formatting.md](formatting.md) + [formatting_wechat.md](formatting_wechat.md)
> 适用场景：通过一个产品/现象，科普一个技术概念的起源、发展与意义

## 内容定义

技术科普不是教程，不是新闻，是**用故事讲清楚一个技术概念为什么重要**。

核心逻辑链：
```
一个产品/现象（钩子）
  → 这背后是什么技术？（概念解释）
  → 这个技术是怎么来的？（起源故事）
  → 它经历了什么？（发展历程）
  → 为什么现在最重要？（核心论点）
  → 它意味着什么？（升华）
```

## 标题公式（三选一）

1. **产品钩子型**：`{产品}终于支持{功能}了，但这背后的东西更有意思`
   - 例：飞书终于支持 Markdown 导出了，但这背后的东西更有意思
2. **概念颠覆型**：`你每天都在用{技术}，但可能不知道它叫什么`
   - 例：你每天都在用 Markdown，但可能不知道它叫什么
3. **人物故事型**：`一个17岁少年和一个博主，改变了整个互联网的写作方式`
   - 例：一个 17 岁少年和一个博主，改变了整个互联网的写作方式

## 结构模板

### 第一层：产品钩子（200-300字）

```
{最近看到/用到} {产品} 的一个更新。

{功能描述}。{为什么这个小功能意义重大}。

{一句话定性：这不只是一个功能，而是…}
```

**要点**：
- 从一个具体产品/新闻切入，不要上来就科普
- 点到即止，不要在产品功能上停留太久
- 引出"这背后的技术"作为过渡

### 第二层：概念解释（300-400字）

```
可能有些朋友看到这，还不太清楚{概念}是什么。

{用读者最熟悉的场景举例}。

比如说，{具体例子}。{这个例子的背后}，其实就是{概念}。

{概念}不是什么高深的技术，它就是{一句话本质定义}。
```

**要点**：
- 假设读者完全不懂，用"可能有些朋友…"开头
- 用读者每天接触但没意识到的场景举例（如"你看 Claude 的回复，有加粗有标题…"）
- 一句话本质定义：不要用学术语言，用大白话
- 这段的目标是让读者产生"哦原来是这个"的顿悟

### 第三层：起源故事（400-600字）

```
{概念}的诞生，得从{年份}说起。

那一年，{人物}遇到了一个{痛点/问题}。

{当时的背景描写}。{他想做什么但遇到了什么障碍}。

{关键洞察/灵感}。{他做了什么}。

他把这个东西叫做{名称}。
```

**要点**：
- **必须有真实人物**：没有人物的故事不是故事，是说明文
- **痛点先行**：先写"他遇到了什么问题"，再写"他怎么解决的"
- **细节感**：年份、地名、工具名、代码示例——越具体越可信
- **名字的由来**：如果有意思，一定要讲（如 Markdown = Mark-down，标记放下来）
- **关键配角**：如果有人物对概念有重要贡献，必须提及（如 Aaron Swartz 与 Markdown）

### 第四层：发展历程（400-600字）

```
{概念}刚出来的时候，{初始状态}。

真正的转折点是{年份}，那一年{关键事件}。

然后是{平台/事件2}。再然后是{平台/事件3}。

{概念}开始从{起点}，逐渐变成了{现在的地位}。
```

**要点**：
- 用时间线串联，但不要写成编年史
- 每个节点只写**一个关键事件 + 一个结果**
- 选择最有戏剧性的转折点（如"GitHub 上线，全世界开发者每天都在写 Markdown"）
- 节奏要快：几句话跳过几年，只留关键节点

### 第五层：核心论点（400-500字）

```
但真正让{概念}封神的，是可能它自己都没想到的一件事。

{新的时代/场景来了}。

{为什么在这个新场景下，这个概念变得不可替代？}

{列举 3-4 个本质原因，每个 1-2 句话}
```

**要点**：
- 这是全文的核心段落，必须有力度
- 用"但真正让 X 封神的"这类句式制造转折
- 原因要从本质出发，不是表面优势
- 如果有数据/对比（如"同样内容，HTML 的 token 比 Markdown 多得多"），效果更好

### 第六层：思辨（300-400字，可选但推荐）

```
{最近有一场争论/一个观点}。

{反方观点}。{说的有没有道理？有。}

但从我的角度，{这个观点混淆了什么/忽略了什么}。

{用一个清晰的框架区分两者}（如"展示 vs 流转"、"数据层 vs 视图层"）

所以{结论}。
```

**要点**：
- 引入反方观点再反驳，比直接论证更有说服力
- 用框架/类比让区分一目了然（"Markdown 是数据层，HTML 是视图层"）
- 承认反方有道理的部分，再指出哪里想偏了
- 不要攻击，要"我觉得他混淆了…"

### 第七层：升华（200-300字）

```
{概念}背后其实是一种哲学/理念。

{创造者/关键人物一生在追求的东西}。

{他做的其他事情，都指向同一个方向}。

{年份}，{人物的结局}。

在他之后，{他创造的东西长成了什么}。
```

**要点**：
- 从技术上升到理念/哲学，但不要空洞
- 用人物的一生来承载理念（如"Aaron Swartz 一辈子追求的：信息的自由流动"）
- 如果人物有悲剧性结局（去世、入狱等），适当提及但不要消费
- 回扣到当前时代的意义

### 第八层：情感收束（100-200字）

```
{回到开头的产品/现象}。

{一句话总结这个概念在这个时代的意义}。

{用一个感性的句子收束}。

就像{年份}，{人物}。{最初想要的那样}。
```

**要点**：
- 回扣开头，形成闭环
- 最后一句要有情感重量
- 用"就像…最初想要的那样"回扣人物
- 不要加"点赞在看转发"之类的功能性结尾混在这段里

### 功能性结尾（独立于情感收束）

```
---

以上，既然看到这里了，觉得不错，随手**点赞 + 在看 + 转发**三连吧。

你对{概念}有什么使用经验或看法？欢迎评论区聊聊。
```

## 语言风格

### 写作原则

| 原则 | 要求 |
|------|------|
| 讲故事，不是写百科 | 有人物、有冲突、有转折 |
| 具体，不要抽象 | "2004 年，John Gruber 在博客上遇到一个问题" > "Markdown 起源于 2004 年" |
| 口语化 | "这玩意拿来写内容太离谱了"、"思路全断了" |
| 允许主观 | "我觉得挺有趣的"、"从我的角度" |
| 节奏感 | 长短句交替，段落有疏有密 |

### 句式模板库

**引入概念**：
- "可能有些朋友看到这，还不太清楚 X 是什么"
- "但其实你只要用 AI，大概率已经每天都在看它了"
- "X 不是什么高深的技术，它就是…"

**起源故事**：
- "那一年，X 遇到了一个很抓狂的问题"
- "X 灵机一动，就把这些散落在…的民间约定，整理成了…"
- "背后甚至任何人都没有，也没有商业模式，就是单纯的觉得…"

**转折与发展**：
- "真正的转折点是{年份}，那一年{事件}"
- "X 开始从…，逐渐变成了…"
- "但真正让 X 封神的，是可能它自己都没想到的一件事"

**思辨**：
- "说的有没有道理，我说实话，有"
- "但从我的角度，我觉得这个观点混淆了两件事"
- "X 和 Y 从来也不是替代关系，是分工关系"

**升华**：
- "X 背后其实是一种哲学"
- "在他去世后的这十几年里，他参与创造的这些东西，全都长成了…"
- "你的 X，就是你的 X。纯粹的，干净的，自由的"

### 禁止使用

- "让我们一起来了解一下"
- "首先…其次…最后…最后…"（机械排列）
- "赋能"、"生态"、"闭环"
- 百科式开头："X 是一种…"、"X，全称…"
- 没有故事的概念解释（概念必须嵌入故事中）

## 结构节奏

全文控制在 **2500-4000 字**：

```
产品钩子（200-300字）
  ↓
概念解释（300-400字）
  ↓
起源故事（400-600字）← 全文最"故事化"的部分
  ↓
发展历程（400-600字）← 节奏加快，几个关键节点
  ↓
核心论点（400-500字）← 全文最"有力度"的部分
  ↓
思辨（300-400字）← 可选，增加深度
  ↓
升华（200-300字）← 全文最"有温度"的部分
  ↓
情感收束 + 功能性结尾（150字）
```

### 写作顺序建议

不要从头写到尾。推荐顺序：
1. 先写**核心论点**（全文的灵魂）
2. 再写**起源故事**（全文的骨架）
3. 然后写**概念解释**（确保读者能跟上）
4. 再写**产品钩子**（根据概念反推最佳切入点）
5. 然后写**发展历程**（串联故事和论点）
6. 最后写**思辨 + 升华 + 收束**（情感加码）
