# 盘点合集类 — 小红书版

> 适配"九卿"人设，发布平台：小红书
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：N个工具/方法的精选清单精简版，30秒让读者知道有哪些好东西

## 与微信版的核心差异

| 维度 | 微信版 | 小红书版 |
|------|--------|---------|
| 目标 | 每个都详细展开 | 30秒知道有哪些+值不值得试 |
| 深度 | 每个300-500字+截图 | 每个1-2句话 |
| 字数 | 2500-4000字 | 500-800字 |
| 结构 | 逐个展开+使用建议 | 清单体，每个一个emoji+一句话 |

## 标题公式（三选一，20字以内）

1. **数字型**：`{N}个{场景}必装的{东西}🔥`
   - 例：6个AI必装的Skills🔥
2. **感受型**：`这{N}个{东西}太好用了😭{一句话}`
   - 例：这3个AI工具太好用了😭回不去了
3. **年度型**：`{年份}最值得装的{N}个{东西}💡`
   - 例：2026年最值得装的5个AI工具💡

## 结构模板

### 开头：3秒钩子（2-3行）

```
{emoji} {一句话说清盘点主题}

{筛选标准/推荐理由}
```

**要点**：
- 第一句说清"我在推什么"
- 点明筛选标准（"我挑了最常用的"）
- 不铺垫，直接进清单

### 主体：N个推荐清单

```
{emoji} {名称}
{一句话说清干嘛的}。{为什么推荐}。

{emoji} {名称}
{一句话说清干嘛的}。{为什么推荐}。

{emoji} {名称}
{一句话说清干嘛的}。{为什么推荐}。
```

**要点**：
- 每个推荐：一个emoji + 名称 + 一句话
- 不展开细节，只给核心信息
- 最推荐的放最前面
- 可以加安装方式（一句话）

### 结尾：互动引导

```
💬 你还有什么好用的{东西}推荐？评论区补充
📌 收藏这篇，需要的时候直接拿
```

## 语言风格

| 原则 | 要求 |
|------|------|
| 精选感 | "我挑了"、"最常用的" |
| 一句话 | 每个推荐不超过两句话 |
| Emoji | 每个推荐1个功能性Emoji |
| 可操作 | 读者看完知道是什么+怎么装 |
| 短句 | 每句不超过25字 |

### 微信 → 小红书简化对照

| 微信版写法 | 小红书版写法 |
|-----------|------------|
| 300字的Frontend Design介绍+截图+安装方式 | "🎨 Frontend Design 解决AI前端的审美问题，Anthropic官方出品" |
| 500字的办公四件套详细介绍 | "📄 办公四件套 word/ppt/excel/pdf 全能读能写，必装" |
| 400字的cc-switch使用教程 | "🔄 CC Switch 一键切换Agent模型，51K星标" |

## 封面图建议

- 尺寸：1080×1440px（3:4竖版）
- 主标题：盘点主题+数量（60-80px粗体）
  - 例："6个必装AI Skills"
- 副标题：筛选背书（30-40px）
  - 例："我亲自试过的精选"
- 配色：工具感（绿/橙/蓝）
- 可以在封面列出所有推荐名称

## 配图清单

- [ ] 封面图（列出所有推荐名称）
- [ ] 每个推荐的产品截图（可选，如果封面已包含可省略）
- [ ] 安装/使用截图（可选）

## 禁止使用

- 超过 30 字的单句
- 连续 3 行没有 Emoji 或加粗
- "让我们来看看"、"今天给大家分享"
- 没有个人体验的纯清单
- 推荐理由太泛（"好用"、"推荐"不算理由）
