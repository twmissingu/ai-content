# AI工具版本更新 — 微信公众号版

> 适配"九卿"人设，发布平台：微信公众号
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：AI工具/模型的版本更新、实测体验、对比评测
> 类型判定：1-2个产品有跑分/价格/对比 → 工具评测类；3个以上产品全面覆盖 → 新闻类

## 内容定义

工具更新不是通稿搬运，是**"我第一时间试了 → 跑了数据 → 有了结论 → 值不值得你折腾"**。

与新闻的区别：新闻是"发生了什么"，工具更新是"我试了，效果如何，你该怎么选"。

核心逻辑链：
```
产品/模型发布（时效钩子）
  → 第一时间接入实测（跑分/数据）
  → 真实体验（Coding/对话/Agent场景）
  → 价格对比（值不值）
  → 适用人群建议（谁该用、谁不该用）
  → 行业意义/展望
```

## 标题公式（四选一）

1. **实测型**：`实测{产品名}，{一句话定性}`
   - 例：实测DeepSeek V4，为国产化而生
   - 例：实测小米MiMo-V2.5-Pro，这可能是目前国内最适合Claude Code的新模型
2. **反差型**：`{产品}{意外特点}，没想到`
   - 例：DeepSeek V4居然放弃了多模态，没想到
3. **对比型**：`{产品A}vs{产品B}，{结论}`
   - 例：DeepSeek V4 vs MiMo-V2.5，国产模型谁更值得用
4. **推荐型**：`{场景}最佳选择，我选{产品}`
   - 例：Claude Code接国产模型，我现在最推荐这个

## 结构模板

### 第一层：时效钩子 + 情绪铺垫（200-300字）

```
{时间}，{产品}终于{发布了/上线了}。

{情绪铺垫：最近模型太多太忙了/等了很久/贤者模式}。

{第一时间做了什么：接入实测/跑数据}。
```

**要点**：
- 强调时效性（"今天"、"昨天凌晨"、"刚刚"）
- 铺垫忙碌感/期待感（"这一周发了7、8个新模型"、"仿佛鸡排哥"）
- 表明"我第一时间试了"——给读者承诺

### 第二层：跑分/数据（400-600字）

```
先看跑分。

{产品自己的跑分数据}。

{跟竞品的对比表格/图表}。

{关键指标解读：哪里强、哪里弱}。
```

**要点**：
- **数据先行**：跑分、参数量、上下文窗口、价格——具体数字
- **对比表格**：跟竞品放一起比，给读者锚点
- **诚实评价**：好就说好，差就说差（"只能说中庸"、"第一梯队水平"）
- **跑分局限**：提醒读者跑分仅供参考（"大家都是赢学"、"大概看一眼就行"）
- **配图必须**：跑分截图、对比表格截图

### 第三层：真实体验（400-600字）

```
然后说说真实体验。

{我在什么场景下用了它}。

{具体case：输入了什么、输出了什么}。

{我的感受：丝滑/拉胯/意外的好}。
```

**要点**：
- **场景具体**：Coding、对话、Agent、写作——明确场景
- **case展示**：真实的输入输出对比，不是空说"效果好"
- **体验诚实**：好就说好，差就说差（"坦率的讲"、"一声长叹"）
- **截图/录屏**：实际使用截图、对比截图
- **跟竞品对比**：同一case在不同模型上的表现

### 第四层：价格分析（200-300字）

```
然后是价格。

{具体价格：输入/输出每百万token}。

{跟竞品的价格对比}。

{性价比评价}。
```

**要点**：
- **具体数字**：输入X元、输出X元每百万token
- **换算美元**：方便跟海外模型对比
- **对比锚点**：Claude Opus、GPT-5.5的价格作为参照
- **性价比判断**：便宜多少、值不值

### 第五层：适用人群建议（200-300字）

```
{产品}适合谁？

{场景1}：{推荐/不推荐}。{原因}。

{场景2}：{推荐/不推荐}。{原因}。

{一句话总结推荐}。
```

**要点**：
- **分场景**：Coding、知识创作、日常对话、Agent
- **明确推荐**：不要含糊，直接说"推荐"或"不推荐"
- **替代方案**：如果不推荐，给替代方案（"用GLM-5.1更好"）
- **一句话总结**：方便读者快速决策

### 第六层：行业意义/展望（200-300字）

```
{从这个产品看行业趋势}。

{技术细节透露了什么信号}。

{未来会怎样}。
```

**要点**：
- 从产品上升到行业（"国产化"、"算力博弈"、"多模态趋势"）
- 技术细节的深层含义（"MXFP4意味着适配国产卡"）
- 展望要克制，不要过度推测

### 功能性结尾

```
---

以上，觉得不错，随手**点赞 + 在看 + 转发**三连吧。

你用{产品}了吗？体验如何？欢迎评论区聊聊。
```

## 语言风格

### 写作原则

| 原则 | 要求 |
|------|------|
| 数据驱动 | 跑分、价格、参数——具体数字 |
| 体验诚实 | 好就说好，差就说差 |
| 对比明确 | 跟竞品放一起比，给读者锚点 |
| 场景具体 | Coding/对话/Agent——明确场景 |
| 吐槽真实感 | "坦率的讲"、"一声长叹"、"有点拉了" |
| 推荐明确 | 直接说"推荐"或"不推荐" |

### 句式模板库

**时效钩子**：
- "{时间}，{产品}终于{发布了}。"
- "等了一年的{产品}，终于发布了。"
- "这一周发了7、8个新模型，我现在就仿佛鸡排哥。"

**跑分引入**：
- "先看跑分。"
- "不过也就大概看一眼就行，反而大家都是赢学。"
- "成绩很不错，跑分上，也比自家之前进步不少。"

**体验评价**：
- "坦率的讲，识别率不太稳定。"
- "丝滑，无敌丝滑。"
- "意外的还不错，非常像那个味道。"
- "便宜大碗省token，效果还很好。"

**价格对比**：
- "大概能便宜60%左右。"
- "换算成美元的话…"
- "作为对比，Claude Opus是…，GPT-5.5是…"

**推荐收尾**：
- "你相信我，此时此刻，用{组合}就是我觉得目前效果最好的组合。"
- "能一步到位就一步到位。"
- "从今天开始，{行动建议}。"

### 禁止使用

- "据悉"、"据了解"（太官方通稿）
- "赋能"、"生态"、"闭环"
- 没有数据的空洞评价
- 全文只夸不批（失去可信度）
- 没有对比的孤立评价

## 结构节奏

全文控制在 **1500-3000 字**：

```
时效钩子 + 情绪铺垫（200-300字）
  ↓
跑分/数据（400-600字）← 含对比表格、截图
  ↓
真实体验（400-600字）← 含实际case、截图
  ↓
价格分析（200-300字）
  ↓
适用人群建议（200-300字）← 明确推荐
  ↓
行业意义/展望（200-300字）
  ↓
功能性结尾（100字）
```
