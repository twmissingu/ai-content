# 技术科普 — 小红书版

> 适配"九卿"人设，发布平台：小红书
> 继承排版规范：[formatting.md](formatting.md) + [formatting_xhs.md](formatting_xhs.md)
> 适用场景：用短平快方式科普一个技术概念，让人 30 秒内 get 到"这东西是什么、为什么重要"

## 与微信版的核心差异

| 维度 | 微信版 | 小红书版 |
|------|--------|---------|
| 目标 | 读完理解整个故事 | 30 秒知道"这是啥+为啥重要" |
| 故事 | 完整起源+人物+发展 | 只保留最惊艳的 1-2 个故事点 |
| 深度 | 有思辨、有升华 | 点到为止，勾起兴趣 |
| 字数 | 2500-4000 字 | 400-800 字 |

## 标题公式（三选一，20字以内）

1. **冷知识型**：`你每天都在用的{技术}，居然是{意外事实}发明的`
   - 例：你每天都在用的 Markdown，居然是一个博客发明的
2. **数字型**：`{技术}只有{N}个符号，却改变了整个互联网`
   - 例：Markdown 只有 10 个符号，却改变了整个互联网
3. **反差型**：`17岁少年发明的东西，现在所有AI都在用`
   - 例：17 岁少年发明的东西，现在所有 AI 都在用

## 结构模板

### 开头：3 秒钩子（3 行以内）

```
{emoji} {一句话冷知识/反差事实}

{用"你可能不知道"引出共鸣}
```

**要点**：
- 第一句话必须让人产生"啊？"的反应
- 不要铺垫，直接抛最炸裂的事实
- 3 行以内进入正题

### 主体：3-5 个要点（每个 2-3 行）

```
{emoji} {要点标题}
{1-2 句话说明}
```

**要点选择原则**：
- 微信版写 7-8 层，小红书版只保留最惊艳的 3-5 个
- 优先级：冷知识 > 故事细节 > 技术原理 > 发展历程
- 每个点独立可读（读者可能只看其中一个就划走）

### 推荐要点结构

```
📌 {概念}是什么
{一句话定义}，{一个你每天接触的例子}。

📖 谁发明的
{年份}，{人物}因为{痛点}发明了它。{最惊艳的一个细节}。

🔥 为什么现在火了
{AI 时代/某个新场景}让它变成了{不可替代的东西}。

💡 一个好玩的事
{最有趣的一个冷知识/轶事}。

✅ 一句话总结
{定性判断}。
```

### 结尾：互动引导（2-3 行）

```
💬 你平时用{概念}吗？评论区聊聊～
📌 收藏这篇，下次有人问你就甩给他
```

## 语言风格

### 写作原则

| 原则 | 要求 |
|------|------|
| 短句 | 每句不超过 25 字 |
| 口语 | "这玩意"、"绝了"、"真的牛" |
| Emoji | 每个要点 1 个功能性 Emoji |
| 简化 | 技术术语用大白话替代 |
| 悬念 | "最牛的是…"、"你猜怎么着" |

### 微信 → 小红书简化对照

| 微信版写法 | 小红书版写法 |
|-----------|------------|
| "Markdown 的诞生，得从 2004 年说起。那一年，一个叫 John Gruber 的博主遇到了一个很抓狂的问题" | "2004 年，一个博客作者受不了写 HTML，发明了 Markdown" |
| "Aaron Swartz，14 岁参与 RSS 1.0 开发，后来联合创办了 Reddit" | "17 岁天才少年 Aaron Swartz 帮忙设计的，他后来创办了 Reddit" |
| "Markdown 是数据层，HTML 是视图层，你不会用视图层来存储数据的" | "Markdown 管结构，HTML 管长什么样，各管各的" |
| "信息的自由流动" | "文字就是文字，不用被格式绑架" |

### 禁止使用

- 超过 30 字的单句
- 连续 3 行没有 Emoji 或加粗
- 学术化定义："X 是一种基于…的标记语言"
- 长段人物生平（保留 1 个最惊艳的细节就够）

## 完整示例

```
💡 你每天都在用的 Markdown，可能不知道它叫这个名字

Claude 回复你的内容，有加粗有标题有代码块对吧。背后的格式，就是 Markdown。

┈┈┈┈┈┈┈┈┈┈┈┈

📌 Markdown 是什么
一套超简单的纯文本规则。两个星号加粗，井号写标题，反引号写代码。不用开 Word，不用学 HTML。

📖 谁发明的
2004 年，一个叫 John Gruber 的博主受不了写 HTML，就整理了一套语法。帮他完善的是 17 岁的 Aaron Swartz——后来 Reddit 的联合创始人。

🔥 为什么 AI 都在用它
纯文本，AI 好生成。有结构，人好阅读。够轻量，省 token。现在 Claude、GPT、Cursor 的底层输出全是它。

💡 一个好玩的事
有人写文章说 HTML 要取代 Markdown，但那篇文章本身是用 Markdown 写的😂 因为 Markdown 的流通性太高了。

✅ 一句话总结
AI 时代保存文件，就用 .md。纯粹、干净、自由。

┈┈┈┈┈┈┈┈┈┈┈┈

💬 你平时用 Markdown 吗？评论区聊聊～
📌 收藏这篇，下次有人问"Markdown 是啥"直接甩给他
```
