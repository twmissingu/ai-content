# 微信公众号排版格式规范

> 继承通用规范 [formatting.md](formatting.md)，本文件仅定义微信公众号特有参数。
> 适用于微信公众号长文（1000-3000字）的排版设计。

## 字号体系

| 元素 | 字号 | 说明 |
|------|------|------|
| 正文 | **15px** | 移动端最佳可读字号，14px 偏小，16px 偏大 |
| 小标题 | 15px 加粗 | 与正文同字号，用加粗 + 颜色区分层级 |
| 大标题 | 18-20px | 仅用于文章主标题 |
| 注释/脚注 | 12-13px | 辅助信息，灰色 |
| 代码/命令 | 14px | 行内代码用灰色底色包裹 |

### 标题层级样式

| 层级 | 字号 | 样式 | 间距 |
|------|------|------|------|
| 一级标题（文章标题） | 18px | 加粗，品牌主色 `#1a1a1a` | — |
| 二级标题（大模块） | 15px | 加粗，左侧 3px 色块（如 `#2F54EB`） | 上方空一行，下方不空行 |
| 三级标题（子模块） | 15px | 加粗，无色块 | 同上 |

## 行距与间距

| 属性 | 推荐值 | 说明 |
|------|--------|------|
| 行间距 | **1.75-2.0 倍** | 微信默认 1.6 偏紧凑，调高舒适度 |
| 字间距 | **1-1.5px** | 最大不超过 2px |
| 段间距 | 空一行 | 段落之间自然换行 |
| 页边距（两端缩进） | **15-20px** | 适配 95% 移动设备 |
| 首行缩进 | **不缩进** | 移动端顶格更整洁，技术内容尤其如此 |

## 配色方案

### 基础色

| 元素 | 色值 | 说明 |
|------|------|------|
| 正文 | `#3f3f3f` | 比纯黑柔和，长时间阅读不刺眼 |
| 小标题 | `#1a1a1a` 或品牌主色 | 与正文拉开层级 |
| 注释/辅助 | `#999999` | 弱化存在感 |
| 分割线 | `#e5e5e5` | 细线，不抢视觉 |
| 引用块背景 | `#f0f7ff`（浅蓝）或 `#fff8f0`（浅橙） | 左侧 4px 竖线 |
| 引用块文字 | `#666666` | 比正文浅，区分层级 |

### 强调色（克制使用）

| 用途 | 色值 | 用法 |
|------|------|------|
| 关键数据/版本号 | `#E74C3C`（红） | 全文不超过 3 处 |
| 操作步骤高亮 | `#FFF3CD`（黄底） | 用于具体操作建议 |
| 品牌主色 | 自定义 | 用于标题、重点句，全文统一 |

## 段落排版

### 段落长度

- 每段 **3-5 行**，不超过 6 行
- 超过 5 行的段落必须拆分
- 单独成段的金句（1 行）可作为节奏调节
- 单句不超过 100 字

### 段落间距

- 段落之间：空一行
- 功能模块之间：用分割线 `---` 隔开
- 小标题前：空一行
- 小标题后：不空行，直接接正文

### 列表格式

- 正文中的列表用 `•`（无序列表）
- 建议/步骤部分用数字编号（1. 2. 3.）
- 列表项不超过 5 条，超过则拆分子分组
- 列表项之间不空行

## 引用块样式

用于"感受"、金句或重要提示：

```
┌─────────────────────────────
│ 左侧 4px 色块竖线（品牌色）
│ 背景：#f0f7ff（浅蓝）
│ 文字：#666666
│ 字号：14px
│ 可搭配引号 ❝
└─────────────────────────────
```

## 卡片式设计

用于产品介绍、步骤说明、要点总结：

- 圆角矩形 `border-radius: 8-12px`
- 轻微阴影 `box-shadow`
- 内边距 `padding: 16-24px`
- 背景色区分：浅灰/浅蓝底色 + 白色卡片

## 配图规范

| 类型 | 尺寸 | 说明 |
|------|------|------|
| 封面图 | 900×500px（16:9） | 重点信息放在中间 70% 区域 |
| 正文插图 | 宽度 677px | 高度不超过 1000px |
| 格式 | PNG | 保证清晰度 |

## 分割线使用

- 功能模块之间：`---`
- 正文与结尾建议之间：`---`
- 不要在每个段落之间都加分割线（过度使用会碎片化）

## 视觉节奏模板

```
开头（100-150字，紧凑段落）
  ↓
加粗金句（1行定性判断）
  ↓
功能1 - 最重磅（300-400字）
  ├─ 紧凑段落（痛点描写）
  ├─ 加粗金句（核心变化）
  └─ 松散列表（技术细节）
  ↓
---
  ↓
功能2 - 次重要（200-300字）
  └─ 同上结构
  ↓
---
  ↓
升级建议（150字，数字编号列表）
  ↓
一句话总结 + 互动（80字）
```

## 微信编辑器实操提示

- 中英文混排必须加半角空格（编辑器不会自动处理）
- 段间距比行高大 5-8px（避免段落粘连）
- 切换富文本/Markdown 前保存草稿（避免样式丢失）
- 图片先压缩至 500KB 以内再上传
