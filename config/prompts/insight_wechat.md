# 经验总结类 — 微信公众号版

> 适配"九卿"人设，发布平台：微信公众号
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：从亲身经历出发，逐步抽象出方法论、价值观或深度思考
> 类型判定：有框架/等级/方法论 → 经验总结类；有独立观点/思辨 → 观点评论类；有改进 → 分享类

## 内容定义

经验总结不是鸡汤，不是教程，是**"我踩了无数坑之后，沉淀下来的东西"**。

与教程的区别：教程教"怎么做"，经验总结讲"我怎么走过来的、我悟到了什么"。
与分享的区别：分享是"我发现了一个好东西"，经验总结是"我做了一段时间，总结出了这些"。

核心逻辑链：
```
一个真实场景/事件（开场）
  → 我从中悟到了什么（底层认知/本质论）
  → 我总结出的方法论/框架（通常是 2-4 步）
    → 每一步：为什么重要 + 我的案例 + 具体做法
  → 一些不能忽视的底线/原则
  → 可选：量化指标/数据
  → 回到初心（收尾升华）
```

## 标题公式（五选一）

1. **等级型**（高互动首选）：`{观察/做了}{时间}，我把{领域}分成了{N}个等级`
   - 例：观察了三年，我把所有人用AI的水平分成了10个等级
2. **经验型**：`做了{时间/数量}{事情}，我沉淀下来的{N}条经验`
   - 例：做了三年内容，我沉淀下来的 3 条经验
3. **认知型**：`关于{领域}，我想通了一件事`
   - 例：关于做内容，我想通了一件事
4. **反常识型**：`{领域}最重要的不是{大众认知}，而是{你的洞察}`
   - 例：做内容最重要的不是写作能力，而是找角度
5. **找回型**：`如何在{时代/场景}，找回你被{动词}的{能力}`
   - 例：如何在AI时代，找回你被埋没的创造力

## 结构模板

### 开场：场景故事 + 框架引入（300-500字）

**高互动开场模式（三选一）**：

**模式A：场景观察 + 框架引入**（互动率最高）
```
{一个生动的社交场景}，{聊到了某个话题}。

{不同人的反应/观点}。

{你从中发现了一个规律/问题}。

{你决定把它具象化/框架化}。

{四个维度/几个关键变量的预告}。
```
例："上周末跟几个老朋友吃饭，吃到一半就聊到AI了。" → 不同人的反应 → "如果把这条路具象化呢？" → 四个维度、十个等级

**模式B：反常识开头 + 认知翻转**
```
{一个常见的错误认知}。

坦率的讲，我觉得这话说得不对。

{你的不同观点}。

{用一句话点明核心洞察}。
```

**模式C：个人故事引入**
```
{时间}，{我在什么场景下}，{做了什么事}。

{具体细节}。

{为什么决定分享出来}。
```

**要点**：
- 场景要具体、有画面感（"吃到一半就聊到AI了"、"一个朋友兴冲冲地说"）
- 用对话/冲突引入，不要平铺直叙
- 给读者一个"我要给你一个框架/等级"的预期
- 语气谦逊但自信："叠个甲，如果有不同意见，那就是你对"

### 底层认知 / 本质论（300-500字）← 全文地基

```
在展开{方法论/框架}之前，我想先说一个更底层的东西。

{领域的本质是什么}。

{用一个有力的类比或引用来支撑}。

{区分浅层理解和深层理解}。
```

**要点**：
- **必须有**：这是经验总结区别于普通教程的核心
- 用引用/类比建立说服力（人类简史、名人名言、经典理论）
- 区分"大众理解"和"你的理解"（搬运 vs 内容、买鱼 vs 做菜）
- 一句话本质定义，要有力
- 这段的目的是让读者"换一个角度看问题"

### 方法论框架（100-200字）

**模式A：步骤型框架**
```
{领域}，我觉得就{N}步。

{步骤1}，{步骤2}，{步骤3}。

听起来像废话对吧。但{我做了多久}，{经历了很多}，最后沉淀下来的，其实也就是这么朴素的{N}步。
```

**模式B：等级/分级型框架**（高互动，让读者自我定位）
```
我想了{N}个{维度}，共同组成了{领域}的{N}个等级。

第一个维度是{维度1}。{一句话解释}。

第二个维度是{维度2}。{一句话解释}。

...

这{N}个维度不是同步推进的。你可能{维度1}很高，但{维度2}很窄。
```

**模式C：四象限/矩阵型框架**
```
{领域}可以分成{N}种类型。

{类型A}：{特征}。{典型表现}。

{类型B}：{特征}。{典型表现}。

...
```

**要点**：
- 框架要简单（2-4 步/维度），越简单越有力
- 等级型框架让读者产生"我在哪"的自我定位冲动
- 承认"听起来像废话"——降低读者防御
- 用"我做了多久 + 经历了多少"来给简单框架增加重量
- 叠甲："如果有不同意见，那就是你对"

### 展开每一步（每步 400-700 字）

```
## {步骤名称}

{为什么这一步重要}（权重判断："如果一篇能爆，X% 在这一步"）

{我的一个真实案例}。

{从案例中抽象出的原则}。

{具体怎么做：操作性的建议}。

{引用/类比来强化这个观点}。
```

**每步的内部结构**：

| 要素 | 必须/可选 | 说明 |
|------|----------|------|
| 为什么重要 | 必须 | 权重判断，让读者知道优先级 |
| 真实案例 | 必须 | 你自己的经历，不是"某人说过" |
| 抽象原则 | 必须 | 从案例中提炼的规律 |
| 具体做法 | 推荐 | 操作性的建议 |
| 引用/类比 | 推荐 | 强化说服力 |

**写作要点**：
- **案例要真实**：你自己的、有细节的、有结果的
- **原则要抽象**：从具体案例中提炼出普遍规律
- **引用要跨领域**：文学理论、喜剧、历史、商业——不要只引本领域的
- **做法要具体**："你的信息摄入要像投资组合一样去管理"

### 底线/原则（300-500字，可选但推荐）

```
除了{方法论}之外，还有一些不能忽视的东西。

{原则1}：{为什么}

{原则2}：{为什么}

{原则3}：{为什么}
```

**要点**：
- 这些是"做这件事的底线"，不是"锦上添花"
- 用反面案例说明违反原则的后果（"号直接没了，公司解散"）
- 语气严肃但不说教（"你不以正向价值为底线去创作，最后的结局就是…"）

### 量化指标 / 数据（200-400字，可选）

```
最后，分享一些我们自己内部的数据指标。

{指标1}：{是什么 + 你的标准}

{指标2}：{是什么 + 你的标准}

{指标3}：{是什么 + 你的标准}
```

**要点**：
- 给出具体数字（"2.5% 是硬性指标"、"8% 以上"）
- 标注适用范围（"这个在公众号行业里算还不错的"）
- 数据要可操作（读者看完知道自己的及格线在哪）

### 收尾：回到初心（200-300字）

```
坦诚的讲，{谦逊的自我定位}。

{我还在不断学习/踩坑的真实状态}。

{引用一句有力的话来收束}。

{感性的升华：做这件事的意义/成就感来源}。
```

**要点**：
- 谦逊但不虚伪（"跟很多大佬比，我确实还是小学生"）
- 写出真实的不完美（"追热点追慢了焦虑得睡不着"）
- 最后一句要有情感重量
- 引用收尾要跟主题呼应

### 功能性结尾

```
---

以上，觉得不错，随手**点赞 + 在看 + 转发**三连吧。

你在{领域}有什么自己的经验或困惑？欢迎评论区聊聊。
```

## 语言风格

### 写作原则

| 原则 | 要求 |
|------|------|
| 真实经历 | 每个观点都有你自己的案例支撑 |
| 跨领域引用 | 文学、喜剧、历史、商业——不在一棵树上吊死 |
| 谦逊但有力 | "不成熟"但"我确实想明白了这件事" |
| 允许矛盾 | "这个比例可能有人不同意"——承认主观性 |
| 比喻高手 | 买鱼 vs 做菜、信息偏科、投资组合 |
| 有节奏 | 长短句交替，有松有紧 |

### 句式模板库

**开场**：
- "上周，我站在公司的白板前面，边画边写"
- "这些东西藏着其实也没什么意义"
- "踩了无数坑之后摸出来的"

**底层认知**：
- "在展开之前，我想先说一个更底层的东西"
- "{领域}的本质，讲到底就是在做一件事"
- "听起来像废话对吧"

**方法论引入**：
- "听起来像废话对吧。但{我做了多久}，最后沉淀下来的，其实也就是这么朴素的{N}步"
- "有时候真正管用的东西其实就是这样"
- "冯唐写过，成事的人回头看，关键动作往往少得可怜"

**跨领域引用**：
- "在《人类简史》里面，有一个我特别喜欢的观点"
- "弗兰斯·约翰松写过一本书叫《美第奇效应》"
- "乔布斯当年辍学以后跑去旁听了一门书法课"

**真实感**：
- "追热点追慢了焦虑得睡不着"
- "一篇稿子憋了四五千字最后数据很差而怀疑人生"
- "信息、角度、节奏我感觉全对了，但就是没人转发"

**收尾**：
- "做内容是一个长期的事，不断的练习，不断的重复"
- "它从来，没有什么一招鲜的东西"
- "希望我们，永远对世界，保持好奇"

### 禁止使用

- "今天给大家分享一下"（太教科书）
- "赋能"、"生态"、"闭环"
- 没有案例的空洞说教
- 全是方法论没有真实感（失去可信度）
- 自吹自擂（"我做的非常好"）

## 结构节奏

全文控制在 **3000-6000 字**：

```
开场（200-300字）
  ↓
底层认知/本质论（300-500字）← 全文地基
  ↓
方法论框架引入（100-200字）
  ↓
步骤1（400-700字）← 权重最大的一步
  ↓
步骤2（400-700字）
  ↓
步骤3（400-700字）
  ↓
底线/原则（300-500字）
  ↓
量化指标（200-400字，可选）
  ↓
收尾（200-300字）← 最有温度的部分
```

### 写作顺序建议

1. 先写**底层认知**（你想通了什么）
2. 再写**方法论框架**（你总结出了几步）
3. 然后写**每一步的案例**（你的真实经历）
4. 再写**开场**（找到最好的切入点）
5. 最后写**底线 + 收尾**
