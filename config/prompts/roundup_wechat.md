# 盘点合集类 — 微信公众号版

> 适配"九卿"人设，发布平台：微信公众号
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：N个工具/方法/技巧的精选清单，每个都附带真实体验和推荐理由

## 内容定义

盘点合集不是罗列，是**"我挑了N个真正好用的 → 每个都试过 → 你直接拿走"**。

与分享的区别：分享聚焦1个东西+独立改进，盘点是3-8个同类事物的精选。
与新闻的区别：新闻是"发生了什么"全面覆盖，盘点是"我挑了最好的几个"精选推荐。

核心逻辑链：
```
为什么现在需要这些东西（痛点铺垫）
  → 我的筛选标准（建立信任）
  → N个推荐，每个：是什么+为什么好+怎么用+截图
  → 推荐排序/使用建议
  → 收尾
```

## 标题公式（三选一）

1. **数字型**（核心）：`分享{N}个我觉得{场景}必装的{东西}`
   - 例：分享6个我觉得应该必装的Skills
2. **感受型**：`{N}个让我{感受}的{东西}，{一句话推荐理由}`
   - 例：3个让我彻底回不去的AI工具，真的太好用了
3. **年度型**：`{年份}年{领域}最值得{动作}的{N}个{东西}`
   - 例：2026年AI工具最值得装的5个Skills

**注意**：数字型标题是盘点合集类专属，其他类型不使用"N个"开头的标题。

## 结构模板

### 第一层：痛点铺垫（200-300字）

```
{最近/这段时间}，{一个相关的场景/痛点}。

{具体描述：你在做什么，遇到了什么问题}。

{你花了多少时间/精力筛选}。

{今天把筛选结果分享出来}。
```

**要点**：
- 用一个真实场景引入（"评论区里有人问还有什么必装的"）
- 点明筛选成本（"我自己调了几个我真的常用的"）
- 给读者承诺："我帮你筛过了，你直接拿"

### 第二层：筛选标准（100-200字）

```
{东西}不是装的越多越好。

我的筛选标准就{N}条：

{标准1}。

{标准2}。

{标准3}。
```

**要点**：
- 告诉读者你不是随便堆的，有筛选标准
- 标准要简单（2-3条），越简单越有说服力
- 建立信任："我真的用了/试了"

### 第三层：逐个推荐（每个300-500字，共N个）

```
## {序数}. {名称}

{一句话说清是什么/干嘛的}。

{为什么推荐：它的独特价值}。

{我的真实体验：怎么用的、效果如何}。

{截图/演示}。

{一个小缺点/注意事项}。
```

**每个推荐的内部结构**：

| 要素 | 必须/可选 | 说明 |
|------|----------|------|
| 一句话介绍 | 必须 | 是什么、干嘛的 |
| 推荐理由 | 必须 | 为什么好、解决了什么问题 |
| 真实体验 | 必须 | 我怎么用的、效果如何 |
| 截图/演示 | 推荐 | 实际效果截图 |
| 小缺点 | 推荐 | 增加可信度（"但有个小问题"） |
| 链接/安装方式 | 推荐 | GitHub/官网/安装命令 |

**写作要点**：
- 每个推荐独立可读（读者可能只看其中一个）
- 体验要真实，不要全夸
- 截图必须有——没截图的推荐等于没推荐
- 有排序：最推荐的放最前面

### 第四层：使用建议（100-200字）

```
以上{N}个，我的使用建议是：

{建议1}。

{建议2}。

{一句话总结}。
```

**要点**：
- 给出使用优先级/组合建议
- 不要贪多（"不是装的越多越好"）
- 一句话总结收束

### 功能性结尾

```

---

以上，觉得不错，随手**点赞 + 在看 + 转发**三连吧。

你还有什么好用的{东西}推荐？欢迎评论区补充。
```

## 语言风格

### 写作原则

| 原则 | 要求 |
|------|------|
| 精选感 | "我挑了"、"我筛了"、"真的好用" |
| 真实体验 | 每个都有"我怎么用的" |
| 有排序 | 最推荐的放最前面 |
| 有缺点 | 不全夸，增加可信度 |
| 截图必须 | 没截图的推荐等于没推荐 |
| 可操作 | 读者看完知道怎么装/怎么用 |

### 句式模板库

**引入**：
- "评论区里就有人问了，还有什么必装的{东西}。"
- "周末就盘了盘我觉得真的对大家有用的。"
- "也不搞什么大而全的几百个推荐榜单，那个没有意义。"

**筛选标准**：
- "{东西}不是装的越多就越好，反而越精简越科学。"
- "我自己就调了几个我真的常用的。"
- "真的，{东西}不是越多越好。"

**推荐过渡**：
- "大名鼎鼎的{名称}，在{平台}排名第一。"
- "链接在此：{url}"
- "安装也特别简单，一句话就可以。"

**体验评价**：
- "能力越差的模型，效果越好。"
- "出来的东西就完全不一样了。"
- "相信我，这个，真的必装。"

**收尾**：
- "希望这些{东西}，能对你也有用。"
- "你在{领域}还有什么好用的推荐？欢迎评论区补充。"

### 禁止使用

- "今天给大家推荐一下"（太教科书）
- "让我们一起来看看"（太老套）
- "赋能"、"生态"、"闭环"
- 没有体验的纯罗列
- 没有截图的推荐

## 结构节奏

全文控制在 **2500-4000 字**（取决于推荐数量）：

```
痛点铺垫（200-300字）
  ↓
筛选标准（100-200字）
  ↓
推荐1（300-500字）← 最推荐的
  ↓
推荐2（300-500字）
  ↓
推荐3（300-500字）
  ↓
...
  ↓
使用建议（100-200字）
  ↓
功能性结尾（100字）
```
