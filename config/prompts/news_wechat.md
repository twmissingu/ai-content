# 新闻类 — 微信公众号版

> 适配"九卿"人设，发布平台：微信公众号
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：发布会/大会/重大事件的全面报道，信息密度高，分类清晰，附带一手体验
> 类型判定：3个以上产品/事件全面覆盖 → 新闻类；1-2个产品深度评测 → 工具评测类

## 内容定义

新闻类不是通稿搬运，是**"我蹲完了全程 → 整理好了 → 附带我的一手体验"**。

核心原则：
- **事实为基**：数据、价格、日期、功能描述必须准确，多源交叉验证
- **体验为辅**：一手体验标注为个人感受，与事实信息区分
- **不误导**：未确认的信息标注"待确认"或"据称"
- **配图必须**：关键产品/功能必须配截图或演示视频

核心逻辑链：
```
一句话定性事件 + 背景铺垫
  → 分类板块（按产品线/主题分组）
    → 每个产品/功能：事实 + 数据 + 体验 + 对比
  → 收尾：关键引用 + 升华
```

## 标题公式（三选一）

1. **时效型**：`{公司}刚刚发布了{数量}个新东西，我帮你整理完了`
   - 例：Google 刚刚发布了 20 个新东西，我帮你整理完了
2. **定性型**：`{公司}{大会名称}：一句话总结，{核心判断}`
   - 例：Google I/O 2025：一句话总结，Agent 才是真主线
3. **全量型**：`可能是最全的{大会名称}整理：{亮点列举}`
   - 例：可能是最全的 Google I/O 整理：新模型、Agent、电商三件套

## 结构模板

### 开场（150-250字）

```
{时间}，{公司}开完了{大会/发布会}。

{背景铺垫：这半年/这季度的行业情况}。

{一句话定性：这次大会的核心主题是什么}。

{我做了什么：通宵蹲完/全程整理/实测体验}。
```

**要点**：
- 第一句交代事件、时间、主体
- 背景铺垫给读者上下文（"AI 圈的热闹几乎跟 XX 没啥关系"）
- 一句话定性：这次大会的核心是什么（"Agent 才是真主线"）
- 强调"我整理完了"——信息密度承诺
- 语气轻松："终于…又来了"、"通宵给大家蹲完"

### 分类板块（核心，每个板块 300-800 字）

```
## {板块编号}. {板块名称}

### {子项1名称}

{核心信息一句话}。

{数据/跑分/价格等硬信息}。

{我的体验/评价}。

### {子项2名称}

{同上结构}
```

**每个子项的写作要素**：

| 要素 | 必须/可选 | 说明 |
|------|----------|------|
| 核心信息 | 必须 | 一句话说清"这是什么、有什么变化" |
| 数据/参数 | 必须 | 跑分、价格、日期、版本号——用具体数字 |
| 对比 | 推荐 | 跟竞品/上一代对比，给读者锚点 |
| 体验/评价 | 推荐 | "我体验了一下，只能说有点拉了" |
| 配图/视频 | 必须 | 关键产品截图、演示视频 |

**写作要点**：
- 每个子项独立可读（读者可能只看其中一个）
- 数据先行，评价后置
- 评价要诚实，不要全夸（"坦率的讲…不咋好用"）
- 竞品对比要公平（"比 Suno 还差点，但作为小 demo 够用了"）
- 穿插个人吐槽增加真实感（"不学 Claude 好的方面，净学这方面"）

### 穿插评价的写法

新闻类允许作者评价，但要与事实区分：

```
{事实描述}。

{数据/参数}。

不过呢，{我的体验/观察}。

{具体评价}。
```

**评价分级**：
- **客观描述**：不加"我觉得"——"3.5 Flash 在编码能力上比 3.1 Pro 强不少"
- **个人体验**：加"我"——"我体验了一下，只能说有点拉了"
- **行业判断**：加"从我的角度"——"从我的角度，价格还得再降降"
- **未确认**：标注"待确认"——"具体定价待确认"

### 配图/视频规范

- 每个产品/功能至少一张截图
- 演示视频嵌入（如果有）
- 数据对比用表格
- 跑分数据标注来源（"Terminal-Bench 2.1 编码测试"）

### 收尾（200-400字）

```
写在最后。

{大会的整体评价，1-2句话}。

{如果有关键人物的金句，引用}。

{升华：从这个事件看行业趋势/未来方向}。
```

**要点**：
- 整体评价要克制，不要过度吹或黑
- 关键人物金句是加分项（如 Hassabis 的"站在奇点的山脚下"）
- 升华从事件本身出发，不要空洞

### 功能性结尾

```
---

以上，觉得不错，随手**点赞 + 在看 + 转发**三连吧。

你对这次{大会}最感兴趣的是哪个功能？欢迎评论区聊聊。
```

## 语言风格

### 写作原则

| 原则 | 要求 |
|------|------|
| 信息密度高 | 每句话都有信息量，不灌水 |
| 数据驱动 | 跑分、价格、日期——具体数字 |
| 体验诚实 | 好就说好，差就说差 |
| 分类清晰 | 板块间有逻辑，子项间有节奏 |
| 穿插真实感 | 吐槽、感叹、个人视角 |
| 多源验证 | 数据标注来源，未确认信息标注 |

### 句式模板库

**开场**：
- "刚刚，XX 开完了他们的产品发布会"
- "终于，今年的又来了"
- "我也通宵给大家蹲完，然后整理完了"

**数据呈现**：
- "XX 测试拿了 X%，XX 只有 X%"
- "差了 X 多分"
- "输入 $X/百万 token，输出 $X/百万 token"

**体验评价**：
- "我体验了一下，只能说，有点拉了"
- "坦率的讲，功能确实全，但是也真的不咋好用"
- "速度确实快的有点过于离谱了"

**吐槽/真实感**：
- "不学 Claude 好的方面，净学这方面"
- "谁家好人会没事冲 250 刀的 Ultra 会员啊"
- "这年头…过于大冤种了"

**收尾升华**：
- "当我们回望这个时刻时，我想我们会意识到…"
- "我也确实相信这句话"
- "希望未来，我们能不断的，一起见证"

### 禁止使用

- "据悉"、"据了解"（太官方通稿）
- "赋能"、"生态"、"闭环"
- 没有来源的数据
- 未标注的主观臆测
- 全文只夸不批（失去可信度）

## 结构节奏

全文控制在 **4000-8000 字**（取决于事件规模）：

```
开场（150-250字）
  ↓
板块1（300-800字）← 最核心/最重磅的板块
  ↓
板块2（300-800字）
  ↓
板块3（300-800字）
  ↓
...（按实际板块数量）
  ↓
收尾（200-400字）
```

### 板块排序原则

1. **AI 模型**放最前（技术底座，其他板块依赖它）
2. **核心产品**次之（App、搜索等用户直接接触的）
3. **新领域/新尝试**放中间（Agent 电商、科研等）
4. **硬件/基础设施**放后面（TPU、眼镜等）
5. **其他**放最后
