# 分享类 — 微信公众号版

> 适配"九卿"人设，发布平台：微信公众号
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：发现一个有意思的概念/方法/工具，分享给读者，附带独立思考和改进
> 类型判定：有"我改进了/我优化了" → 分享类；无改进但有框架 → 经验总结类；有跑分/价格 → 工具评测类

## 内容定义

分享类不是搬运，是**"我发现了一个好东西 → 我试了 → 我想明白了 → 你也值得知道"**。

与科普的区别：科普是"讲故事让你理解"，分享是"我有感悟想跟你说"。

核心逻辑链：
```
我遇到了一个问题/场景（个人故事）
  → 我发现了一个东西（来源交代）
  → 这个东西是什么 + 为什么有效（介绍）
  → 但我觉得还不够（独立思考）
  → 我做了这些改进（优化展示）
  → 跑一遍看效果（实际演示）
  → 这意味着什么（升华）
```

## 标题公式（四选一）

1. **发现型**（核心）：`我发现了一个{形容词}的{东西}，{一句话说清价值}`
   - 例：我发现了一个帮助理解概念的神器
2. **工具型**：`这个{N}K星标的开源神器，让{场景}变得{效果}`
   - 例：这个51K星标的开源神器，让任何Agent都能一键切换所有模型
3. **体验型**：`试了{东西}之后，我觉得{观点}`
   - 例：试了用 AI 讲寓言故事之后，我觉得这是最快乐的学习方式
4. **时效型**：`{时间}，{产品}终于{动作}了`
   - 例：豆包输入法的Mac版，终于正式上线了
   - 例：我发现了一个帮助理解概念的神器
4. **体验型**：`试了{东西}之后，我觉得{观点}`
   - 例：试了用 AI 讲寓言故事之后，我觉得这是最快乐的学习方式
5. **时效型**：`{时间}，{产品}终于{动作}了`
   - 例：豆包输入法的Mac版，终于正式上线了

## 结构模板

### 第一层：反差钩子开头（200-300字）

**高互动开头模式（三选一）**：

**模式A：反差/悬念开头**（推荐，互动率最高）
```
{一句话抛出反常识/震撼的观点}。

{铺垫：为什么这么说}。

{个人故事引入}。
```
例："GPT-image-2上线后，整个互联网都变成了巨大的黑暗森林。" → 展开猜疑链 → 引出信任话题

**模式B：场景故事开头**
```
{时间}，{我在什么场景下}，{发生了什么事}。

{具体细节描写}。{我的感受/困惑}。

{从故事中提炼出一个普遍问题}。
```
例："上周末跟几个很久没见的老朋友吃饭，吃到一半就聊到AI了。" → 不同人的反应 → 引出分级想法

**模式C：等待释放开头**
```
{产品/工具}终于{上线/发布了}。

我等这一天，等了好久好久。

{为什么等这么久}。{为什么现在可以写了}。
```

**要点**：
- 第一句话必须有钩子——反差、悬念、或情感共鸣
- 场景要具体：谁、在哪、做了什么、发生了什么
- 要有真实的困惑/不完美
- 这段的目的：让读者产生"然后呢？"的好奇心

### 第二层：问题提炼（100-200字）

```
{具体的问题}。

到底有没有什么办法，能{理想的解决状态}？

然后，我就想到了{来源}。
```

**要点**：
- 从个人故事中提炼出一个具体问题
- 问题要读者也能感同身受（"怎么让大家真正记住一个概念"）
- 过渡到"我发现了一个东西"——自然引出

### 第三层：发现来源（100-150字）

```
这个方法/概念，来自{人物}，是{他/她}{时间}在{来源}里分享的。

{人物}可能很多人不太了解，{一句话背景介绍}。
```

**要点**：
- 交代来源：谁、什么时候、在哪（播客、文章、访谈）
- 给人物一个简短背景（"Claude 性格对齐团队的负责人"）
- 不要长篇介绍人物，点到即止

### 第四层：原版介绍（300-500字）

```
{他/她}的思路是这样的。

{核心方法/概念的一句话解释}。

{详细展开：原版是什么样的}

{为什么有效：你的分析}
```

**要点**：
- 先给一句话概括，再展开
- 如果有原版 Prompt/方法/框架，**直接展示原文**
- "为什么有效"是你的独立分析，不是复述
- 用"我觉得"、"从我的角度"表达个人观点

### 第五层：独立思考 / 改进（400-600字）← 全文核心

```
然后我在{他/她}的基础上，{优化/改进/思考了一下}。

主要是因为{发现了什么问题}。

{问题1}：{具体描述}

{问题2}：{具体描述}

所以我在原版的基础上做了几个改动。

{改动1}：{为什么改 + 改成什么样}

{改动2}：{为什么改 + 改成什么样}
```

**要点**：
- **这是全文最有价值的部分**——你的独立思考
- 问题要具体："AI 编故事容易走向类似的叙事结构"，不是"有一些不足"
- 改动要说明**为什么**改，不只是改成什么样
- 如果有参考别人的好补充，也提一句
- 最终给出完整的优化版（Prompt/方法/框架），可复制

### 第六层：实际演示（400-600字）

```
比如说，{用一个具体概念跑一遍}。

{展示过程/结果}

{你的感受}。
```

**要点**：
- 用 1-2 个真实例子演示
- 展示实际效果（引用输出内容）
- 给出你的感受："鸡皮疙瘩都起来了"
- 如果有多个例子，用不同领域展示通用性

### 第七层：扩展应用（200-300字，可选）

```
除了{原始场景}，还可以{扩展用法}。

{具体例子}
```

**要点**：
- 展示 1-2 个扩展用法（如连环画、其他领域）
- 不要展开太多，点到即止

### 第八层：引用/背书（200-300字）

```
{原创者/专家}在{来源}里聊到{为什么做这个}的时候说了一段话。

{引用内容}

{你的感受/评价}。
```

**要点**：
- 引用原创者或权威人士的话
- 如果有访谈/播客，引用主持人的回应也很好
- 不要长篇引用，选最有力量的 1-2 段
- 引用后给出你的评价

### 第九层：升华（200-300字）

```
{从这个具体的东西，上升到更大的视角}。

{历史/哲学/人性层面的思考}。

{用一个感性的句子收束}。
```

**要点**：
- 从技术/方法上升到人性/哲学
- 用历史类比增强说服力（"佛经是故事，圣经是故事，庄子也是故事"）
- 最后一句要有情感重量

### 功能性结尾

```
---

推荐大家都可以去试试。

以上，觉得不错，随手**点赞 + 在看 + 转发**三连吧。

你有什么{概念/方法}想用这个方式试试？欢迎评论区留言。
```

## 语言风格

### 写作原则

| 原则 | 要求 |
|------|------|
| 真实发现感 | "我发现了"、"我试了"、"我感觉" |
| 边想边说 | "然后我就想到"、"说到这我真心觉得" |
| 有缺陷的诚实 | "没讲明白"、"还是似懂非懂" |
| 独立思考 | "我在她基础上优化了"、"我觉得还不够" |
| 类比高手 | 用读者熟悉的事物解释新东西（游戏、三体、马具等跨领域类比） |
| 引用具体 | 有名有姓有出处，不是"有人说" |
| 金句密度 | 每300-500字至少一句可截图传播的金句 |
| 吐槽真实感 | "真的，体验就差太多了"、"坦率的讲"、"一声长叹" |
| 数据锚点 | 具体数字增强说服力（"51K星标"、"10万UV"、"60%便宜"） |

### 句式模板库

**反差钩子**：
- "因为{产品}，整个{领域}都变成了巨大的{比喻}。"
- "{产品}终于{动作}了。我等这一天，等了好久好久。"
- "今天想写个有趣的东西。"

**场景引入**：
- "上周末正好出差，跟几个很久没见的老朋友吃饭"
- "昨天，我把我的{产品}向大家免费开放了。"
- "今天没选题了，所以想开源一个我自己做的东西。"

**真实感铺垫**：
- "我自己已经内测使用了快1个月了"
- "得到了很多很多朋友的喜欢。也万万没想到"
- "在一个通宵之后，我终于全部开发完了。"

**产品介绍过渡**：
- "网址在此：{url}"
- "链接在此：{url}"
- "你下载打开以后，直接安装。"

**体验评价**：
- "坦率的讲，识别率不太稳定"
- "丝滑，无敌丝滑。"
- "真的很强，你会发现，当你用的越久，就会越用越准"

**独立思考**：
- "主要是因为我自己跑多了发现几个问题"
- "然后我在她基础上，优化了一下"
- "所以我做了几个改动"

**升华/收尾**：
- "从今天开始，张嘴说话吧。相信我，别再跟你的键盘，较劲了。"
- "过期的就删，重复的就合，模糊的就改。让你的知识库，永远只保留此刻最准确的真相。"
- "希望，我能给这个互联网，留下一点点，自己的印记。"
- "真诚是唯一的捷径。这并不所谓的鸡汤。这是我自己，在黑暗森林里的生存策略。"

### 禁止使用

- "今天给大家分享一个…"（太教科书）
- "让我们一起来看看"（太老套）
- "赋能"、"生态"、"闭环"
- 没有来源的"据说"、"有人说"
- 纯搬运没有自己的思考

## 结构节奏

全文控制在 **2500-4000 字**：

```
反差钩子/场景故事（200-300字）← 第一句话决定打开率
  ↓
问题提炼（100-200字）
  ↓
发现来源（100-150字）
  ↓
原版介绍（300-500字）← 含原始 Prompt/方法/工具链接
  ↓
独立思考/改进（400-600字）← 全文最有价值的部分
  ↓
实际演示（400-600字）← 含截图、对比、数据
  ↓
扩展应用（200-300字）
  ↓
引用/背书（200-300字）
  ↓
金句升华（200-300字）← 可截图传播的收尾
  ↓
功能性结尾（100字）
```

### 高互动文章特征

基于大量高赞文章分析，以下特征显著提升互动率：

| 特征 | 说明 | 示例 |
|------|------|------|
| 反差开头 | 第一句话制造认知冲击 | "整个互联网都变成了巨大的黑暗森林" |
| 数据锚点 | 具体数字增强说服力 | "51K星标"、"10万UV"、"便宜60%" |
| 截图/对比 | 装前vs装后、改进前vs改进后 | 不装Skill的效果 vs 装了之后的效果 |
| 金句密度 | 每300-500字一句可截图传播的话 | "当造假的成本趋近于零，信任的成本就趋近于无穷大" |
| 产品链接 | 文末附工具/代码/Prompt下载 | GitHub链接、官网链接 |
| 吐槽真实感 | 诚实评价不好用的地方 | "坦率的讲，识别率不太稳定" |

### 写作顺序建议

1. 先写**独立思考/改进**（你最想说的）
2. 再写**原版介绍**（提供上下文）
3. 然后写**个人故事**（找最好的切入点）
4. 再写**演示**（验证你的思考）
5. 最后写**升华 + 收束**
