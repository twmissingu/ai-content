# 经验总结类 — 小红书版

> 适配"九卿"人设，发布平台：小红书
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：从亲身经历提炼 3-5 条经验，一句话说清每条，让人收藏

## 与微信版的核心差异

| 维度 | 微信版 | 小红书版 |
|------|--------|---------|
| 目标 | 读完理解你的思考深度 | 30 秒 get 到几条有用的经验 |
| 深度 | 有底层认知、跨领域引用、详细案例 | 只保留最有力量的一句话经验 |
| 字数 | 3000-6000 字 | 400-700 字 |
| 结构 | 长叙事 + 方法论 + 案例 | 条目式，每条 2-3 行 |

## 标题公式（三选一，20字以内）

1. **经验型**：`做了{时间}{事情}，这{N}条经验最值钱`
   - 例：做了三年内容，这 3 条经验最值钱
2. **认知型**：`关于{领域}，我想明白了一件事💡`
   - 例：关于做内容，我想明白了一件事💡
3. **反常识型**：`{领域}最重要的不是{大众认知}，而是这个`
   - 例：做内容最重要的不是写作，而是找角度

## 结构模板

### 开头：3 秒钩子（2-3 行）

```
{emoji} {一句话核心洞察}

{背景：我做了多久/经历了什么}
```

**要点**：
- 直接抛最有力量的洞察
- 背景一句话就够（"做了三年内容，踩了无数坑"）

### 主体：3-5 条经验（每条 2-3 行）

```
{emoji} {经验标题}
{1-2 句话说明}，{一个关键词或比喻}
```

### 推荐经验结构

```
📖 {本质论}
{领域}的本质是{一句话}。{一个有力的比喻}。

🔍 {经验1：最重要的一条}
{具体经验}。{为什么}。

💡 {经验2}
{具体经验}。{一个案例的一句话概括}。

📊 {经验3：可量化的}
{具体标准}。{及格线是多少}。

⚠️ {底线}
{不能碰的东西}。{违反的后果}。
```

### 结尾：互动引导

```
💬 你在{领域}有什么自己的经验？评论区分享
📌 收藏这篇，迷茫的时候翻出来看看
```

## 语言风格

| 原则 | 要求 |
|------|------|
| 金句化 | 每条经验要能独立截图传播 |
| 比喻 | 用一个比喻让经验更生动 |
| 短句 | 每句不超过 25 字 |
| Emoji | 每条 1 个功能性 Emoji |
| 真实感 | "踩了无数坑"、"焦虑得睡不着" |

## 完整示例

```
💡 做了三年内容，我最想明白的一件事：内容的本质是讲故事

不是搬运资讯，不是翻译论文，是讲故事👇

┈┈┈┈┈┈┈┈┈┈┈┈

📖 本质
做内容 = 讲故事。公众号、短视频、小红书，底层都是讲故事。搬运信息和做内容的差距，就像买鱼直接扔给别人 vs 做成菜端上桌🐟

🎯 找角度最重要
一篇能不能爆：找角度 60%，获取信息 30%，写作 10%。好角度 = 情理之中 + 预料之外。你第一直觉想到的别写，因为所有人都能想到。

🌐 信息要跨领域
只看本领域 = 信息偏科。综艺学叙事、历史学厚度、生活学人性。你永远不知道哪个领域的知识会在什么时候帮到你。

📊 我们的数据标准
赞阅比 2.5% 及格，转阅比 8% 以上才算好内容，完读率 40%（2000字）。比阅读量更能告诉你内容好不好。

⚠️ 底线
不懂的别说，敏感的别碰，别装。违反的后果：号没了，公司解散。

┈┈┈┈┈┈┈┈┈┈┈┈

💬 你做内容最深的体悟是什么？评论区聊聊
📌 收藏这篇，做内容迷茫的时候翻出来
```

## 封面图建议

- 尺寸：1080×1440px（3:4 竖版）
- 主标题：核心洞察（60-80px 粗体）
  - 例："内容的本质是讲故事"
- 副标题：背书（30-40px）
  - 例："做了三年，踩了无数坑"
- 配色：温暖感（不要太科技感）

## 配图清单（小红书版）

- [ ] 封面图（决定点击率）
- [ ] 核心洞察金句图（可截图传播）
- [ ] 方法论/框架图（如果有分级/矩阵）
- 建议 3-4 张图

## 禁止使用

- 超过 30 字的单句
- 连续 3 行没有 Emoji 或加粗
- "让我们来看看"、"今天给大家分享"
- 纯鸡汤（每条经验必须有具体支撑）
- 没有数据或案例的空洞总结
