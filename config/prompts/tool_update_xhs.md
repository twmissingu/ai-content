# AI工具版本更新 — 小红书版

> 适配"九卿"人设，发布平台：小红书
> 人设规范：[persona_bible.md](persona_bible.md)
> 适用场景：AI工具/模型版本更新的精简报道，挑最值得说的点

## 与微信版的核心差异

| 维度 | 微信版 | 小红书版 |
|------|--------|---------|
| 目标 | 全面评测，数据充分 | 30秒知道值不值得试 |
| 深度 | 跑分+体验+价格+行业 | 只说最关键的一两个点 |
| 字数 | 1500-3000字 | 300-600字 |
| 配图 | 跑分截图、对比表 | 封面图+核心对比图 |

## 标题公式（三选一，20字以内）

1. **实测型**：`实测{产品}，{一句话结论}🔥`
   - 例：实测DeepSeek V4，国产模型终于能打了🔥
2. **对比型**：`{产品}vs{产品}，谁更值得用？`
   - 例：DeepSeek V4 vs MiMo，谁更值得接Claude Code？
3. **推荐型**：`{场景}最佳选择，我选这个💡`
   - 例：Claude Code接国产模型，我现在最推荐这个💡

## 结构模板

### 开头：3秒钩子（2-3行）

```
{emoji} {产品}终于{发布了}！

{一句话定性：值不值得试/最让我惊喜的点}
```

### 主体：3-5个要点

```
🏃 {跑分/性能}
{一句话结论}，{关键数据}。

💰 {价格}
{具体价格}，比{竞品}便宜{X}%。

⚡ {真实体验}
{一个具体case的感受}。

🎯 {适用人群}
{谁该用/谁不该用}。

🔮 {行业意义}
{一句话展望}。
```

### 结尾：互动引导

```
💬 你用{产品}了吗？体验如何？
📌 收藏这篇，选模型的时候翻出来看
```

## 语言风格

| 原则 | 要求 |
|------|------|
| 直接 | 不铺垫，直接说结论 |
| 感叹 | "真的猛"、"没想到"、"有点拉" |
| Emoji | 每个要点1个功能性Emoji |
| 数据 | 关键数字必须有 |
| 短句 | 每句不超过25字 |

### 微信 → 小红书简化对照

| 微信版写法 | 小红书版写法 |
|-----------|------------|
| "DeepSeek V4 在 Terminal-Bench 2.1 编码测试拿了 76.2%，3.1 Pro 只有 70.3%" | "新模型编码能力直接碾压上一代 Pro" |
| "API 价格从 $15/M tokens 降到 $6/M tokens，降幅 60%" | "API 价格直接砍了 60%，真的猛🔥" |
| 500 字的使用场景分析 | "写代码首选这个，写文章用另一个" |
| 详细跑分对比表格 | 一张对比图 + 一句话结论 |

## 封面图建议

- 尺寸：1080×1440px（3:4竖版）
- 主标题：产品名+核心结论（60-80px粗体）
  - 例："DeepSeek V4 / 为国产化而生"
- 副标题：最亮眼的一个数据（30-40px）
  - 例："比Claude便宜60%"
- 配色：产品品牌色

## 配图清单（小红书版）

- [ ] 封面图（决定点击率）
- [ ] 跑分对比图（一张图说清楚）
- [ ] 真实使用截图（代码/对话效果）
- [ ] 价格对比图（如果价格有变化）
- 建议 3-5 张图

## 禁止使用

- 超过 30 字的单句
- 连续 3 行没有 Emoji 或加粗
- "让我们来看看"、"今天给大家分享"
- 纯跑分堆砌（要有结论）
- 没有个人体验的纯参数罗列
